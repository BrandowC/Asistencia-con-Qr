Asistencia con QR

Sistema académico para tomar asistencia con sesiones temporales, código QR y código de sala anti-fraude. Esta landing está diseñada para Moodle como paquete SCORM y resume la documentación técnica, funcional y de sustentación sin publicar datos sensibles reales.

Problema que resuelve Asistencia académica sin papel ni mezclas

En una clase presencial o híbrida, tomar asistencia de forma manual consume tiempo, dificulta la trazabilidad y aumenta el riesgo de errores o registros duplicados.

Tiempo operativo

Inicio rápido

El docente o instructor abre una sesión y comparte el QR sin interrumpir la clase.

Trazabilidad

Auditoría

La sesión guarda inscritos, presentes, ausentes, rechazos, duplicados e historial.

Control académico

Por institución

Asistencia organizada por institución, ficha o materia. Sin mezcla de unidades.

Privacidad: en Moodle no se publican nombres reales, documentos, teléfonos, correos, contraseñas, tokens ni URLs activas. Todo en este paquete usa datos sintéticos.

Flujo funcional para clase

  1. Inicio de sesión: docente o instructor entra con un usuario demo.
  2. Selección académica: elige institución y unidad académica.
  3. Inscritos: revisa la lista de estudiantes/aprendices.
  4. Activación QR: crea sesión, activa el QR temporal y muestra el código de sala.
  5. Registro público: el estudiante usa el QR o link para registrarse.
  6. Resultados: presentes, ausentes, rechazos e historial.

Instalación reproducible para estudiantes Windows / PowerShell + Docker

git clone https://github.com/BrandowC/Asistencia-con-Qr.git
cd Asistencia-con-Qr
Copy-Item .env.example .env
npm run install:all

Levantar servicios

npm run docker:up
npm run docker:seed
Invoke-RestMethod http://localhost:4000/health
Invoke-RestMethod http://localhost:4000/ready
npm run dev:app

Flujo de consumo frontend

async function login(documento, password) {
  const response = await fetch(`${backendUrl}/api/auth/login`, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ documento, password })
  });
  if (!response.ok) {
    throw new Error('Credenciales inválidas o API no disponible');
  }
  return response.json();
}

Validación de QR

Reglas esperadas:
1. El token debe existir.
2. La sesión debe estar activa.
3. El QR no debe estar expirado.
4. El documento debe pertenecer a la unidad académica.
5. No debe existir asistencia accepted previa para misma sesión/persona.
6. Si falla, registrar rechazo con motivo trazable.

Checklist antes de subir a Moodle

Uso como SCORM en Moodle

Este paquete incluye imsmanifest.xml en la raíz, index.html, estilos y JS separados.

  1. Entrar al curso.
  2. Activar edición.
  3. Agregar actividad o recurso.
  4. Seleccionar paquete SCORM.
  5. Subir el ZIP completo.
  6. Guardar y mostrar.
Importante: Moodle necesita el ZIP completo. No subir solo index.html. imsmanifest.xml debe quedar en la raíz del paquete.
